Step 1 — Drug Representation

Why Hypergraphs? SMILES isn't enough.

A SMILES string encodes a molecule as a linear text. It contains all the atoms and bonds, but it's a 1-D sequence — it can't natively represent the structural relationships that matter for binding.

❌ SMILES String

Cn1nc(C(F)(F)F)c2c(=O)
c3cc(Cl)ccc3n(O)c21

⚠️

Limitations

Linear sequence — same molecule has multiple valid SMILES. Rings and multi-atom environments aren't first-class. CNNs / LSTMs must learn structure from scratch.

✅ Hypergraph

● Bond edges ● Ring hyperedges ● Env hyperedges
🔗

Multi-way Relationships

A hyperedge can connect 3, 4 or more atoms simultaneously — e.g. an entire aromatic ring is ONE hyperedge. SMILES can't do this.

🌀

Ring Awareness Built In

Ring membership is an explicit structural hyperedge — the model knows aromaticity before any training step.

🌐

Local Chemical Environment

Each atom gets an "environment" hyperedge covering its 1-hop neighbourhood — capturing functional-group context efficiently.

🚀

SMILES-Invariant

The hypergraph is built from the RDKit graph, not the string. It's the same structure regardless of which SMILES notation you write.

Why Distillation?

The 650M teacher has seen vastly more protein data and captures richer evolutionary context. By blending its frozen CLS into the student's sequence, the student's cross-attention can leverage 650M-level protein understanding with only 35M compute cost at inference.

Cosine similarity (student vs teacher after alignment) = 0.999998

→ L_distill = 1 − 0.999998 = 0.000002 (nearly identical!)
